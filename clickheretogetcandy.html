Click Here to Get Candy

The world’s first AI powered candy dispensing system.

Overview

Click Here to Get Candy is an interactive art installation that was constructed over the course of one month. I was tasked with working on both the physical component and the web server. My primary responsibilities included setting up Arduino motors to dispense candy, designing interaction touchpoints, and programming the front-end and back-end of the web server.

Problem

We began this project with the idea of spreading awareness towards biased algorithms that are used in artificial intelligent machines today. After conducting research, we soon realized that many companies utilize these algorithms to automate many tasks such as hiring new job candidates, approving people for bank loans or schools, and many more things. The problem with this approach is that data is often polluted by other insignificant factors such as gender and race.

Interaction

After completing our research, we decided on creating a machine learning candy dispenser that would only give candy to participants that passed our biased algorithm. We planned to mimic current AI algorithms used in society today to raise awareness on the issue.

We chose to use candy as a reward for the childhood semantics revolving around the concept of candy, as in a sweet reward that is given to those who deserve it.

Participants will be judged on this decision by the following criteria: Name, Sex, Mood, Age, and Random Number Selection. The weighing of the questions varies. Some answers to questions will have a larger impact on whether that person receives candy or not. While other questions will have little to no impact on the decision. During peer feedback and critiques, we were advised that adding a random/irrelevant factor makes the testing more interesting, and makes the determining factors much less evident. We followed this advice to create a more immersive interactive experience for our audience.

Physical Component

In order to dispense the candy automatically I decided to use Arduino to communicate serially with our web server. When a participant would pass our algorithm, our server would send a signal to Arduino to rotate its motors to dispense one piece of candy.

Web Server

As we needed to be able to easily communicate serially with Arduino, we decided to construct our web server using Processing, which has a built-in library to work with Arduino. For the machine to decide which participants to accept or reject, I created an “ideal user model” that contained data points on possible values for age, sex, mood, and random number. Once the participant provided their answers, JavaScript was utilized to evaluate their credentials against our model. Finally, the post request handler would send a value to Arduino to spin its motors if the participant’s credentials were accepted.

To extract the age, sex, and mood of the participant I utilized a face analyzing API that would get this information from a picture of the participant’s face. Similar to our “ideal user model” that was used to make the decision on whether a participant would get candy or not, this face API has several models to intelligently guess a users age, sex, and mood.

Reflection

This artwork highlights the potential dangers of misused technology, i.e. how an algorithm would inherit the values and ideology of its creator. Future refinements for this project would include improving the physical art piece to look aesthetically pleasing. This project has helped develop my programming abilities in languages such as Processing, Arduino, HTML, CSS, and JavaScript. Furthermore, this project gave me the opportunity to examine different varying interactions a user can have with an interface and how I, as a designer, can create these experiences for the user.
